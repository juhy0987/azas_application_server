# Code Convention

이 문서는 `projects_display` 저장소의 기본 코드 컨벤션입니다.

## 1. Python 스타일

- Python 버전: `3.12+`
- PEP 8 준수
- 들여쓰기: 공백 2칸
- 최대 줄 길이: 100자 권장
- 타입 힌트 사용 권장 (공개 함수/메서드는 필수)
- 문자열은 기본적으로 쌍따옴표(`"`) 사용

## 2. FastAPI 규칙

- 라우터 함수는 동사보다 리소스 중심으로 명명
- API 경로는 소문자 kebab-case 또는 snake_case 중 하나를 고정해서 사용
- 응답 스키마는 가능하면 Pydantic 모델로 명시
- 비즈니스 로직은 라우터에서 분리

## 3. 파일 구조 규칙

- `main.py`: 앱 엔트리포인트 및 라우팅 등록
- `templates/`: HTML 템플릿
- `static/`: CSS, JS, 이미지
- `data/`: 샘플/로컬 데이터
- 새 기능 추가 시 디렉터리 목적을 벗어나지 않도록 유지

## 4. 네이밍 규칙

- 변수/함수: `snake_case`
- 클래스: `PascalCase`
- 상수: `UPPER_SNAKE_CASE`
- 템플릿 파일명: 소문자 + 하이픈 또는 언더스코어

## 5. 주석과 문서화

- 코드가 충분히 명확하면 불필요한 주석 금지
- 복잡한 의도, 예외 처리, 제약사항 중심으로 짧게 작성
- 공개 함수는 간단한 docstring 권장

## 6. 의존성 관리

- 의존성은 `pyproject.toml`로만 관리
- 새 라이브러리 추가 시 목적을 PR 설명에 포함
- 사용하지 않는 의존성은 제거

## 7. 품질 체크

PR 전 최소 점검:

- 서버 실행 확인 (`uvicorn main:app --reload`)
- 주요 경로 수동 테스트 (`/`, `/api/projects`)
- 템플릿/JS 콘솔 에러 확인

## 8. 커밋 단위

- 하나의 논리적 변경은 하나의 커밋으로 구성
- 리팩터링과 기능 추가를 같은 커밋에 섞지 않기
- 커밋 메시지는 변경 의도를 명확하게 작성
